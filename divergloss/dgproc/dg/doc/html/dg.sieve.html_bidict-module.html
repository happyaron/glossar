Module html_bidict

Create HTML table with bilingual dictionary.

For example, having a bilingual English/Serbian (en/sr) glossary gloss.xml, the HTML page with the dictionary table could be created by:

   $ dgproc.py html-bidict gloss.xml -s olang:en -s tlang:sr \ 
                                     -s file:gloss.html -s style:igloo

The olang parameter specifies the original language for the dictionary, and tlang the target language. The file parameter names the file of the resulting HTML page, as well as the basename for the accompanying style sheet and JavaScript control files included by the HTML page header. The style parameter selects one of the internal style sheets (which are listed in the sieve help, using --help-sieve option). The given command will ultimately produce three files: gloss.html, gloss.css, and gloss.js.

The HTML page will contain only a single compact two-column table, with the left column containing distinct terms from the original language, and the right column the corresponding terms in the target language. If, for a given original term, there are several target terms, and all of them name name the same concept, the target terms will be given as a comma-separated list in the right column. On the other hand, if the original term names several different concepts, and there are different target terms for them, such target terms will be given one below the other in the right column, as an enumerated list.

The description and other details on the concepts behind each term in the target language will be provided as folded sections of the table, which can be unrolled with a +/- button somewhere in the table row. If there several concepts are named by the same term, the details of all of them are given in the folding section for that term.

The style parameter is optional; if not given, the page will be left without a style sheet. This is handy when the user has prepared an own style sheet (which can be based on one of the internal sheets); in such case, the user will probably also make use of the header parameter.

Using the parameters header and footer, the user can specify text files to be prepended before and appended after the HTML code for the table (without them, the sieve provides minimal header and footer to make a valid standalone HTML page).

The user can explicitly specify names for the accompanying CSS and JS files to be created, using the cssfile and jsfile parameters, instead of these names being derived from the name of the HTML file. Again, this is mostly useful in conjunction with external header and footer.

Instead of creating separate CSS and JS files, using phpinc parameter both the style sheet and control functions can be written into a single file, wrapped in appropriate tags for PHP inclusion into the body of the HTML file. The file name of this file too will be derived from the HTML file name as gloss.inc, but an explicit name can be given by the incfile parameter. Note that in this case the file name of the page (given by file parameter) should probably end in .php.

Finally, both the CSS and JS content can be directly embedded into the page, so that only this one HTML file is produced by the sieve. This is done by specifyng the allinone parameter. Note that it conflicts all parameters which deal with auxiliary files for inclusion.

If the glossary contains several environments, one of them may be selected by the usual env parameter. If not given, the default environment is used. Only those concepts which have at least one term in the original and target language and environment will be used as sources of terms.

Internal style sheets (as selected by the style parameter) contain some values which can be tuned through the command line using the styleopt parameter. It takes a comma-separated list of name=value pairs. For example, to set the width of the column with terms in the original language to a third of the table width, use:

   $ dgproc.py html-bidict ... -sstyleopt:'oterm_col_width=33%'

Available style options are listed in the sieve help (-H), under the description of the styleopt parameter.
